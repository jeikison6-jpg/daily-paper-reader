## 摘要
本文介绍了基于知识图谱的大规模多任务基于模型的策略优化（KG-M3PO），这是一个在部分可观测环境下统一了感知、知识和策略的多任务机器人操纵框架。该方法通过在线 3D 场景图增强了第一视角视觉，将开放词汇检测落地为度量式的关系表示。动态关系机制在每一步更新空间、包含和可供性（affordance）边，并且图神经网络编码器通过强化学习（RL）目标进行端到端训练，使得关系特征直接由控制性能塑造。多种观测模态（视觉、本体感受、语言和基于图的）被编码到一个共享的潜空间中，RL 智能体在此基础上运行以驱动控制循环。策略以轻量级图查询以及视觉和本体感受输入为条件，为决策提供了一个紧凑且具有语义信息的决策状态。在一系列包含遮挡、干扰物和布局变化的操纵任务上的实验表明，该方法相比强基线模型取得了持续的提升：知识条件化智能体实现了更高的成功率、更高的样本效率，以及对新物体和未见场景配置更强的泛化能力。这些结果支持了这样一个前提，即结构化、持续维护的世界知识是实现可扩展、可泛化操纵的强大归纳偏置：当知识模块参与 RL 计算图时，关系表示与控制相对齐，从而在部分可观测性下实现稳健的长程行为。

## Abstract
This paper introduces Knowledge Graph based Massively Multi-task Model-based Policy Optimization (KG-M3PO), a framework for multi-task robotic manipulation in partially observable settings that unifies Perception, Knowledge, and Policy. The method augments egocentric vision with an online 3D scene graph that grounds open-vocabulary detections into a metric, relational representation. A dynamic-relation mechanism updates spatial, containment, and affordance edges at every step, and a graph neural encoder is trained end-to-end through the RL objective so that relational features are shaped directly by control performance. Multiple observation modalities (visual, proprioceptive, linguistic, and graph-based) are encoded into a shared latent space, upon which the RL agent operates to drive the control loop. The policy conditions on lightweight graph queries alongside visual and proprioceptive inputs, yielding a compact, semantically informed state for decision making.   Experiments on a suite of manipulation tasks with occlusions, distractors, and layout shifts demonstrate consistent gains over strong baselines: the knowledge-conditioned agent achieves higher success rates, improved sample efficiency, and stronger generalization to novel objects and unseen scene configurations. These results support the premise that structured, continuously maintained world knowledge is a powerful inductive bias for scalable, generalizable manipulation: when the knowledge module participates in the RL computation graph, relational representations align with control, enabling robust long-horizon behavior under partial observability.

### 论文总结：基于知识引导的多任务强化学习操纵框架 (KG-M3PO)

#### 1. 核心问题与研究背景
在非结构化的日常环境中，机器人操纵面临严重的**部分可观测性（Partial Observability）**问题，如物体被遮挡、置于容器内或因光照变化导致视觉失效。传统的纯反应式策略（如像素到力矩的映射）难以处理长程任务中的状态混淆，而传统的模型化方法又过于依赖脆弱的手工几何管线。
本研究旨在通过集成**结构化世界知识（3D 场景图）**、**多任务强化学习框架**以及**端到端表示学习**，弥合感知与策略之间的鸿沟，使机器人能够在复杂、动态且部分观测的环境中实现稳健的操纵。

#### 2. 方法论：KG-M3PO 框架
核心思想是将感知、知识和策略统一在一个计算图中，利用动态更新的 3D 场景图为 RL 智能体提供持久的信念状态。
*   **在线 3D 场景图 (3DSG)：** 基于 BBQ 框架构建，节点包含物体的 3D 边界框、中心点及 CLIP 语义嵌入；边表示空间关系（如“在……之后”、“在……之上”）、包含关系和可供性。
*   **动态更新机制：** 每 10 个步骤局部刷新受影响的子图，通过自我中心流检测物体移动，确保知识的时效性与低计算开销。
*   **端到端图编码：** 使用图神经网络 (GNN) 将场景图编码为紧凑的知识向量 $k_t$。关键创新在于 **GNN 编码器直接参与 RL 损失函数的反向传播**，这意味着关系特征是根据控制性能（而非辅助监督任务）进行优化的。
*   **多模态融合：** 将视觉（腕部相机）、本体感受、语言目标和图知识向量融合进共享潜空间，驱动基于 M3PO（大规模多任务模型化策略优化）的控制循环。

#### 3. 实验设计
*   **实验场景：** 基于 NVIDIA Isaac Lab 构建的物理仿真环境，具有光线追踪渲染和物理一致性。
*   **任务设置：**
    *   **全可观测 (FO)：** 抓取方块 (Pick-Cube)、放置方块 (Pick-Place)、开柜子 (Open-Cabinet)。
    *   **部分可观测 (PO)：** 抓取墙后的物体 (PO Pick)、移除遮挡物后抓取并放置 (PO Pick-Place)。
*   **机器人平台：** Franka Emika Panda (7-DoF) 和 UR5 (6-DoF)。
*   **对比基准 (Baselines)：** 包括 PPO、SAC、DreamerV3、TD-MPC2、IMPALA 以及不带知识图谱的 M3PO（纯视觉）。

#### 4. 资源与算力
*   **硬件设施：** 使用单台工作站，配置为 **NVIDIA RTX 5070 Ti (16 GB VRAM)** GPU 和 64 GB RAM。
*   **训练效率：** 利用 Isaac Lab 的平铺渲染 API (Tiled Rendering) 实现大规模并行，单进程同时运行 **1024 个并行环境**。
*   **训练时长：** 400M 步训练约需 0.61 到 0.79 个 GPU 天（GPU-days）。

#### 5. 实验数量与充分性
*   **实验规模：** 涵盖了 5 种核心任务，并在两种不同的机器人平台上进行了验证。
*   **实验维度：** 进行了单任务对比、多任务联合训练对比、以及“纯视觉 vs. 视觉+知识图谱”的消融对比。
*   **鲁棒性测试：** 专门设计了针对场景图噪声（中心点偏移、尺寸误差、旋转误差）的消融实验，验证了系统在几何不确定性下的退化表现。
*   **评价：** 实验设计较为全面，通过归一化得分（Normalized Score）和成功率进行量化，对比了当前主流的多种 RL 算法，具有较高的客观性和说服力。

#### 6. 主要结论与发现
*   **知识图谱的必要性：** 在部分可观测任务中，纯视觉策略几乎无法成功，而 KG-M3PO 能够通过持久的结构化状态解决状态混淆。
*   **样本效率提升：** 即使在全可观测任务中，引入场景图也能显著加快收敛速度并提高最终成功率。
*   **端到端优势：** 将 GNN 纳入 RL 损失计算图，使得学习到的关系表示更符合控制需求，优于分离训练的特征。
*   **多任务泛化：** 单个策略能够同时处理多种异构任务，证明了融合表示的强大表达能力。

#### 7. 优点与亮点
*   **闭环集成：** 成功将高层语义/关系知识（场景图）与底层连续控制（RL）在端到端框架下结合。
*   **动态性：** 提出了在线关系更新机制，能够处理物体位置变动，而非依赖静态地图。
*   **抗噪性：** 实验证明该方法对一定程度的检测和姿态估计误差具有鲁棒性。
*   **工程实现：** 充分利用了现代 GPU 仿真器的矢量化能力，实现了高效的大规模训练。

#### 8. 不足与局限
*   **Sim-to-Real 缺位：** 实验完全在仿真环境中进行，尚未在真实物理机器人上验证，真实世界的感知噪声和延迟可能更复杂。
*   **计算开销：** 引入场景图模块增加了每一步的计算负担（虽然通过提升样本效率抵消了部分影响）。
*   **环境动态性限制：** 虽然能处理物体移动，但对于极高频变动或拓扑结构剧烈变化的场景（如流体、软体）可能仍有局限。
*   **依赖感知质量：** 尽管有抗噪性，但如果底层物体检测器完全失效，场景图的构建将崩溃。
